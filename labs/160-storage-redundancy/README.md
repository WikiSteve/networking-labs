# Storage Redundancy

> **Before you start:** Download the [Storage Redundancy Submission Template](<./assets/Storage Redundancy SUBMISSION TEMPLATE.pptx>). Add each required screenshot directly into this file as you complete the lab, then submit the completed template for grading.

## Goal
Practice both hardware RAID and software RAID workflows by building RAID 6, RAID 10, RAID-Z2, and mirror storage layouts, then documenting the resulting arrays and pool status.

## Prereqs
- Basic familiarity with RAID terminology and fault-tolerance concepts
- VMware Workstation access for creating or modifying virtual machines
- Enough local storage to host multiple lab disks
- Ability to access a browser for controller software or TrueNAS management

## Deliverables
- Screenshot 1: RAID 6 array creation with the required disks, hot spare, and volume name
- Screenshot 2: RAID 10 array creation with the required disks and volume name
- Screenshot 3: TrueNAS disk set, OS disk, and hostname configured correctly
- Screenshot 4: RAID-Z2 summary with six data drives and one hot spare
- Screenshot 5: mirror pool status showing the four 30 GB drives
