# NetBus Lab

> **Before you start:** Download the [NetBus Lab Submission Template](<./assets/NetBus Lab SUBMISSION TEMPLATE.pptx>). Add each required screenshot directly into this file as you complete the lab, then submit the completed template for grading.

## Goal
Examine an older NetBus remote-access trojan scenario, review the networking concepts it depends on, and use Windows command-line tools to verify listening ports, changed ports, and an active connection.

## Prereqs
- Instructor-provided Windows 7 lab VM
- Instructor-provided Windows Server 2008 lab VM
- NetBus lab files from your instructor
- Basic familiarity with Windows command-line tools such as `netstat`, `find`, and `tasklist`

## Deliverables
- Complete the required challenge screenshot
- Use the submission template linked above for this lab
- Show the changed listening port, the active connection, the password prompt window, and the hostname in the same capture
